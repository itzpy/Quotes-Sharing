# 📝 QuoteShare

A modern, responsive React application for sharing and discussing inspirational quotes. Built with React, Tailwind CSS, and Supabase for a seamless user experience with dark/light mode support.

![QuoteShare Demo](https://via.placeholder.com/800x400/4F46E5/FFFFFF?text=QuoteShare+Demo)

## ✨ Features

- 🔐 **Admin Authentication** - Secure login for quote management
- 💬 **Public Comments** - Anyone can comment without logging in
- 🌓 **Dark/Light Mode** - Toggle between themes with persistent preferences
- 📱 **Responsive Design** - Works beautifully on all devices
- ⚡ **Real-time Updates** - Powered by Supabase for instant data sync
- 🎨 **Modern UI** - Clean, intuitive interface with smooth animations
- 💾 **Persistent Storage** - All data stored securely in Supabase
- 👤 **User Management** - Simple admin authentication system

## 🔧 Installation

1. **Clone the repository**
   ```bash
   git clone https://github.com/itzpy/Quotes-for-Nat.git
   cd Quotes-for-Nat
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Set up environment variables**
   Create a `.env` file in the root directory:
   ```env
   REACT_APP_SUPABASE_URL=your_supabase_project_url
   REACT_APP_SUPABASE_ANON_KEY=your_supabase_anon_key
   CI=false
   ```

4. **Start the development server**
   ```bash
   npm start
   ```

5. **Open your browser**
   Navigate to [http://localhost:3000](http://localhost:3000)

## 🗄️ Supabase Setup

### 1. Create a Supabase Project

1. Go to [supabase.com](https://supabase.com) and create an account
2. Click "New Project" and fill in your project details
3. Wait for the project to be provisioned

### 2. Get Your Credentials

1. In your Supabase dashboard, go to Settings → API
2. Copy your Project URL and anon/public key
3. Update your `.env` file with these values

### 3. Set Up Database Schema

Run the following SQL in your Supabase SQL Editor (`supabase_schema.sql`):

```sql
-- Table for quotes
CREATE TABLE quotes (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  text TEXT NOT NULL,
  author VARCHAR(255) NOT NULL,
  posted_by VARCHAR(255) NOT NULL,
  user_id UUID REFERENCES auth.users(id),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  likes_count INTEGER DEFAULT 0
);

-- Table for comments
CREATE TABLE comments (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  quote_id BIGINT REFERENCES quotes(id) ON DELETE CASCADE NOT NULL,
  content TEXT NOT NULL,
  user_name VARCHAR(255) NOT NULL,
  user_id UUID REFERENCES auth.users(id),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Enable Row Level Security
ALTER TABLE quotes ENABLE ROW LEVEL SECURITY;
ALTER TABLE comments ENABLE ROW LEVEL SECURITY;

-- RLS Policies
CREATE POLICY "Anyone can read quotes" ON quotes FOR SELECT USING (true);
CREATE POLICY "Authenticated users can insert quotes" ON quotes FOR INSERT TO authenticated WITH CHECK (true);
CREATE POLICY "Anyone can read comments" ON comments FOR SELECT USING (true);
CREATE POLICY "Anyone can insert comments" ON comments FOR INSERT USING (true);
```

### 4. Create an Admin User

1. In Supabase dashboard, go to Authentication → Users
2. Click "Invite User" or "Add User"
3. Enter your admin email and password
4. This account will be used for adding new quotes

## 🚀 Deployment

### Deploy to Vercel (Recommended)

1. **Connect to Vercel**
   - Push your code to GitHub
   - Connect your repository to Vercel
   - Vercel will automatically deploy on every push

2. **Environment Variables**
   In your Vercel dashboard, add:
   ```
   REACT_APP_SUPABASE_URL=your_supabase_url
   REACT_APP_SUPABASE_ANON_KEY=your_supabase_key
   CI=false
   ```

3. **Custom Build Command** (if needed)
   ```bash
   CI=false npm run build
   ```

### Manual Deployment

1. **Build the project**
   ```bash
   npm run build
   ```

2. **Deploy using Vercel CLI**
   ```bash
   npx vercel --prod
   ```

## 🐛 Troubleshooting

### Build Issues
- Ensure `CI=false` is set in your environment to prevent treating warnings as errors
- Check that all dependencies are properly installed
- Verify Supabase credentials are correct

### Authentication Issues
- Confirm admin user exists in Supabase Authentication
- Check that Row Level Security policies are properly configured
- Verify Supabase URL and keys are correct

### Theme Issues
- Clear localStorage if theme toggle isn't working
- Ensure Tailwind CSS is properly configured for dark mode
